  name: string

  email: string
}

let user = User {
  name: "Alice",

  email: "alice@example.com"

### v0.3.3 (Planned - Enums)
```kode
enum Status {
  Pending
  Active
  Completed
}

let s = Status::Active
match s {
  Status::Active => print("running")
  _ => print("other")
}
```

### v0.4 - Concurrency
```kode
fn main() {
  let result = chan.new()
  
  spawn {
    result <- 42
  }
  
  let val = <-result
  print(val)
}
```

## 🎓 Key Learnings

1. **Stack-based VMs are excellent for simplicity** - Clean opcode design
2. **Type inference needs careful semantics** - Coercion rules matter
3. **Comprehensive examples are critical** - For adoption and clarity
4. **Testing early prevents major refactors** - Bytecode + runtime both tested
5. **Documentation should evolve with code** - 10+ docs kept up-to-date

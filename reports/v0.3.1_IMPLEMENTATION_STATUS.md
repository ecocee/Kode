## 🏗️ Architecture Notes

### Member Access Implementation

**Parser:** Handles dot notation in postfix expressions
```
obj.member → MemberAccessExpr{Object: obj, Member: "member"}
```

**Typer:** Type-checks member access
- Arrays: .len → FunctionType returning int
- Structs: .field → field type
- Enums: .variant → enum variant

**Compiler:** Generates bytecode
- Compiles object expression
- Emits OpMemberAccess with member name

**VM:** Executes member access
- Pops object from stack
- Looks up member method/field
- Pushes result or method reference

**Runtime:** Evaluates member access
- For arrays: returns length or method reference
- For structs: returns field value
- For enums: returns variant

## Performance Notes

- Array.len: O(1) - Direct stack operation
- Member access: O(1) - Constant time lookup
- Type checking: Minimal overhead
- Bytecode size: +2-3 bytes per member access

## Known Limitations

- `.push()` and `.pop()` return method references but not yet callable
- Struct methods not yet supported
- Enum pattern matching not yet implemented
- No generic types yet
- No trait/interface system yet
